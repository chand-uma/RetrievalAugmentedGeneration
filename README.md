# Text Chunking Strategies

Chunking source documents is crucial because how you split the text greatly affects the quality of retrieval and AI responses.

## Three Main Chunking Methods

### 1. Size-Based Chunking

* Splits text into equal-length chunks.
* Often uses overlapping chunks to maintain context.
* Simple and widely applicable.
* Can cut off words, sentences, or ideas.

### 2. Structure-Based Chunking

* Uses document structure such as:

  * Headers
  * Sections
  * Paragraphs
* Creates more meaningful chunks.
* Requires well-formatted documents.

### 3. Semantic-Based Chunking

* Groups related sentences using Natural Language Processing (NLP).
* Creates chunks based on meaning rather than size.
* More advanced and context-aware.
* More complex to implement.

## Choosing a Chunking Strategy

The best chunking strategy depends on:

* Document format
* Content structure
* Retrieval requirements
* Application needs

**Size-based chunking** is generally the most broadly applicable approach.

# Text Embeddings

Text embeddings are numerical representations of the meaning contained in text. They are generated by embedding models and help AI systems understand and compare text semantically.

## Key Concepts

* An embedding consists of a vector of numbers.
* Each number can be thought of as a score representing some characteristic of the text.
* The exact meaning of individual dimensions is generally unknown.
* Similar text produces similar embeddings.

## Benefits of Embeddings

Embeddings enable:

* Semantic search
* Similarity comparison
* Context retrieval
* Improved AI-generated responses

Rather than matching exact keywords, embeddings help identify text with similar meaning.

# The Full RAG Pipeline

The Retrieval-Augmented Generation (RAG) pipeline consists of multiple stages that work together to provide accurate and context-aware responses.

## RAG Workflow

1. Chunk source text into smaller sections.
2. Generate embeddings for each chunk.
3. Store embeddings and associated text in a vector database.
4. Convert the user's query into an embedding.
5. Retrieve the most relevant chunks.
6. Supply retrieved context to the Large Language Model (LLM).
7. Generate a response.

## Why Store Original Text?

Embeddings alone do not provide human-readable context.

A vector database should store:

* Embedding vectors
* Original chunk text
* Metadata

This allows the system to return meaningful information during retrieval.

## Retrieval Process

The retrieval stage uses similarity measures such as:

* Cosine Similarity
* Dot Product
* Euclidean Distance

These methods identify the chunks most relevant to a user's query.

# BM25 Lexical Search

Semantic search alone may occasionally return less relevant results.

Combining semantic search with a lexical search method such as **BM25** improves retrieval accuracy.

## How BM25 Works

BM25 ranks documents using:

* Term frequency
* Inverse document frequency
* Document length normalization

Rare but important terms receive greater weight.

## Benefits

* Better keyword matching
* Improved precision
* Strong handling of domain-specific terminology

## Hybrid Search

The video demonstrates implementing both:

* Semantic Search
* BM25 Lexical Search

The results are then merged to leverage the strengths of both approaches.

### Advantages

* More precise retrieval
* Better context awareness
* Improved AI-generated responses

This is a foundational technique for advanced AI-powered applications.

## Reciprocal Rank Fusion (RRF)

Results from both indexes are combined using **Reciprocal Rank Fusion (RRF)**.

### Benefits of RRF

* Balances rankings from multiple search systems
* Reduces bias toward any single retrieval method
* Improves overall search relevance

## Outcome

The hybrid retrieval architecture delivers:

* More accurate search results
* Better relevance
* Improved retrieval quality

by leveraging both semantic and lexical search techniques.

# Reranking Results

The video introduces a re-ranking step that further improves retrieval accuracy.

## How Reranking Works

1. Perform semantic search.
2. Perform BM25 search.
3. Merge results.
4. Send the results to Claude.
5. Claude reorders the results based on relevance to the user query.

## Benefits

* Higher retrieval precision
* Better context selection
* Improved RAG performance

## Efficiency Optimization

Instead of sending full document text to Claude:

* Document IDs are sent in the prompt.
* Full text is retrieved only when necessary.

This approach:

* Reduces latency
* Lowers token usage
* Maintains strong retrieval quality
